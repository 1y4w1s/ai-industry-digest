## 一、深色模式收尾（建议优先级：高）

### 问题
深度使用后零星发现硬编码白色/浅色背景未被 CSS 变量覆盖。

### 排查清单

| 区域 | 状态 | 修复方式 |
|------|------|---------|
| 登录页表单输入框背景 | ✅ | `#F8F9FA` → `var(--color-bg-off)` |
| AI 对话气泡（底部浮动） | ✅ | `#FFFFFF` → `var(--color-bg-white)` |
| 设置页按钮（深色下选中态） | ✅ | `#FFFFFF` → `var(--color-bg-white)` |
| Select 下拉菜单 | ⬜ | box-shadow 深色下可见性待检查 |
| 搜索结果页 AI 推荐面板 | ✅ | 已全部替换 |

### 验收标准
- 切换到深色模式后，上述区域背景均变暗
- 无突兀的白色方块残留
- 按钮/输入框文字对比度 > 4.5:1

## 三、文章朗读 TTS（建议优先级：中）

### 功能描述
在文章详情页添加「朗读」按钮，使用浏览器 `SpeechSynthesisUtterance` API 将正文转为语音。

### 实现方案

```
ArticleReader.jsx 新增：
  状态: speaking (boolean), paused (boolean)
  按钮: 朗读 / 暂停 / 停止
  逻辑:
    1. 获取文章正文（纯文本）
    2. 创建 SpeechSynthesisUtterance 实例
    3. 设置中文语音（lang='zh-CN'）
    4. 朗读/暂停/停止控制
```

### 注意事项
- 中文 TTS 质量取决于浏览器/系统语音库
- Chrome 和 Edge 中文朗读效果最好
- 需要处理超长文章的分段朗读（分段避免卡顿）
- 退出阅读页时自动停止

### 验收标准
- 点击「朗读」后开始朗读当前文章正文
- 可暂停和恢复
- 退出文章页时自动停止
- 按钮状态清晰：朗读中 / 已暂停 / 已停止

## 七、AI 对话体验优化（建议优先级：高）

### 问题
首页 AI 助手没有今日日报上下文，用户问"今天有什么新闻"只能回复陈旧知识。

### 修复方案
- 首页对话（无 article_id）时，自动注入最新日报的文章列表作为上下文
- 新增 `DAILY_CONTEXT_PROMPT` 模板，直接告诉 AI"以上是今天的日报"
- 限制注入前 10 篇文章的标题+摘要，控制 token 消耗

### 验收标准
- 问"今天有什么大新闻" → 直接列出今日日报中高重要性文章
- 问"今天 AI 领域有什么值得关注的" → 基于日报内容回答
- 不干扰文章详情页的对话体验

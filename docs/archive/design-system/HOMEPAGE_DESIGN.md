# Signal — 首页设计文档

> 版本：v3.0-homepage
> 基于 V3_REDESIGN.md 的首页部分独立细化
> 设计方向：极简黑白 · 编辑品质 · 财经媒体气质

## 一、页面定位

### 核心功能

| 功能 | 说明 |
|------|------|
| **每日简报** | 默认展示最新一期日报，支持切换历史日期 |
| **文章筛选** | 按重要性/来源/标签过滤当前日报文章 |
| **文章阅读** | 点击文章 → 进入阅读模式（左原文 + 右AI精读+对话） |
| **趋势概览** | 右侧面板：热点词云 + 趋势速览 + Top 5 + 来源分布 |

### 非首页功能

| 功能 | 归属 |
|------|------|
| 全局关键词搜索 | Header 搜索框 → 独立搜索结果页（全屏） |
| 收藏/历史/统计 | 后续新增页面 |
| AI 全局对话 | 右下角黑色气泡 |

**自然语言映射规则：**
| 相对今天 | 显示标签 |
|---------|---------|
| 0 天前（当天） | "今天" |
| 1 天前 | "昨天" |
| 2 天前 | "前天" |
| 3 天前及以上 | "MM/DD 周X" |

**折叠逻辑：**
- 默认展示最近 3 个日期（今天/昨天/前天 或最近 3 期）
- 超过 3 个时，右侧显示「展开所有日期 ▾」按钮
- 点击后展开全部，按钮变为「收起 ▴」
- 选中日期高亮：底部 2px 黑色下划线 + 文字 `#1A1C1E`
- 未选中：文字 `#686C72`，无下划线

**统计信息：**
- 无 emoji，纯文字
- 格式：`23 篇文章 · 4 个来源 · 7 篇高重要性`
- 对齐：右对齐或左对齐（与标题行布局协调）

## 四、状态管理

### 4.1 数据加载流程

```
页面加载
  ↓
请求 /api/reports?page=1&page_size=7
  ↓
获取日报列表 → 默认选中最新一期
  ↓
请求 /api/reports/{report_date}
  ↓
渲染：Overview + Hero + Article List + Right Panel
```

### 4.2 空状态（后端关闭时）

策略：**使用 localStorage 缓存最近一次成功加载的数据**

```js
// 每次成功获取数据后缓存
const cacheKey = 'signal_home_cache';
localStorage.setItem(cacheKey, JSON.stringify({
  reports: data,
  timestamp: Date.now()
}));

// API 失败时读取缓存
const cached = localStorage.getItem(cacheKey);
if (cached) {
  const { reports, timestamp } = JSON.parse(cached);
  // 显示缓存数据 + 顶部提示条
  // 「上次更新: 2小时前 · 数据可能不是最新」
}
```

| 状态 | 展示 |
|------|------|
| **首次加载（无缓存）** | 居中加载动画 |
| **API 失败 + 有缓存** | 显示缓存数据 + 顶部黄色提示条 |
| **API 成功** | 正常展示 + 更新缓存 |
| **无数据 + 无缓存** | 「暂无日报数据」居中提示 |

### 4.3 搜索模式

搜索归 Header 管理，点击搜索图标或回车时：
1. Header 搜索框输入关键词 → 回车
2. 跳转到 `/search?q=xxx`（新路由，非首页）
3. 搜索结果页：全屏，无右侧面板，无日期导航
4. 搜索框保留关键词，支持筛选

此页面后续开发，首页不做内联搜索。

### 4.4 筛选模式

筛选只在当前日报范围内过滤，不调用 `/api/articles` 搜索接口：

```
不筛选 → 展示日报全部文章
选「高」 → 只展示 importance: high 的文章
选「arXiv」 → 只展示 source_name: arXiv 的文章
选「标签」 → 只展示 tags 包含该标签的文章
```

**当前实现已经是调用了 `/api/articles`，这与「在当前日报内过滤」的意图不一致。** 建议修改为前端过滤（filter 数组），而非 API 请求，因为：

- 日报数据已经包含了所有文章，无需再次请求
- 避免加载动画闪烁
- 实现也更简单

### 4.5 日期切换

| 操作 | 行为 |
|------|------|
| 点击日期按钮 | 重新请求 `/api/reports/{date}`，更新整个内容区 |
| 默认 | 选中最新一期（列表第一个） |
| 动画 | 内容区 fadeIn 300ms |

## 九、首页不包含的功能

| 功能 | 原因 |
|------|------|
| 内联搜索 | 搜索是独立体验，不应混合在日报浏览中 |
| 收藏/历史/统计 | 后续页面 |
| 文章分享/导出 PDF | 放在阅读模式的操作栏 |
| 多日期对比 | 用户每次只看一期日报 |
| 自定义排序 | 固定按重要性排序，不提供额外选项 |

## 十、技术实现备忘

### 数据缓存实现

```js
// api/client.js — 新增缓存逻辑
const CACHE_KEY = 'signal_home_cache';
const CACHE_DURATION = 30 * 60 * 1000; // 30 分钟

async function getReportsWithCache(page) {
  try {
    const data = await api.getReports(page);
    localStorage.setItem(CACHE_KEY, JSON.stringify({
      data, timestamp: Date.now()
    }));
    return { data, fromCache: false };
  } catch {
    const cached = localStorage.getItem(CACHE_KEY);
    if (cached) {
      const { data, timestamp } = JSON.parse(cached);
      const age = Math.floor((Date.now() - timestamp) / 60000);
      return { data, fromCache: true, age };
    }
    throw new Error('无可用数据');
  }
}
```

### 筛选优化（前端过滤）

```js
// Home.jsx — 筛选改为前端过滤
const filteredArticles = articles.filter(a => {
  if (importance && a._imp !== importance) return false;
  if (source && a.source_name !== source) return false;
  if (tag && !(a.tags || []).includes(tag)) return false;
  return true;
});
```

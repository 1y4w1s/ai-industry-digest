## 二、性能优化（建议优先级：高）

### P-1.1 首页 API 合并

**现状**：首页加载一次发 6 个请求（reports / sources / tags / reports/{date} / articles/{id} / bookmarks / history）

**方案**：新增 `/api/home` 接口，一次性聚合返回首页所需所有数据

| 当前 | 优化后 |
|------|--------|
| 6 次 HTTP 往返 | **1 次** HTTP 往返 |
| 总等待 ~600ms（串行瓶颈） | ~150ms |
| 前端 6 个 useState + useEffect | 1 个 hook 消费 |


**后端**（`api/routes/content.py`）：
```python
@router.get("/home", tags=["首页"])
async def get_home(user_id: str = Depends(get_user_id)):
    """首页聚合接口：一次返回所有首页数据"""
    reports = db.get_reports(page=1, page_size=7)
    sources = db.get_sources()
    tags = db.get_tags()
    stats = {"total_articles": db.get_article_count()}
    # 如果有最新日报，一并返回详情的文章列表
    report_detail = None
    if reports.get("items"):
        latest = reports["items"][0]
        report_detail = db.get_report_by_date(latest["report_date"])
    return {
        "reports": reports,
        "sources": sources,
        "tags": tags,
        "stats": stats,
        "report_detail": report_detail,
    }
```

**前端**（`hooks/useReport.js`）：
```javascript
async function fetchHome() {
  const data = await api.getHome();
  setReports(data.reports);
  setSources(data.sources);
  setTags(data.tags);
  setReport(data.report_detail);
}
```

### P-1.2 文章阅读缓存

**现状**：每次点开文章，发送 getArticle / getBookmarks / addHistory / getHistory 等多个请求

**方案**：
- 合并 bookmarks 查询：阅读器挂载时一次性获取所有 bookmarks（已有 localStorage 缓存）
- 延迟写入 history：关闭阅读器时批量写入，而非进入时就写
- 使用 `useMemo` 缓存 `stripHtml()` / `renderArticleContent()` 等计算密集型操作

### P-1.3 localStorage 缓存策略升级

**现状**：裸存数据，无过期机制

**方案**：统一缓存层 `utils/cache.js`：

```javascript
export const Cache = {
  set(key, data, ttlMinutes = 60) {
    const item = { data, expires: Date.now() + ttlMinutes * 60000 };
    localStorage.setItem(key, JSON.stringify(item));
  },
  get(key) {
    const raw = localStorage.getItem(key);
    if (!raw) return null;
    const item = JSON.parse(raw);
    if (Date.now() > item.expires) { localStorage.removeItem(key); return null; }
    return item.data;
  }
};
```

| 缓存项 | TTL | 影响 |
|--------|-----|------|
| sources | 24h | 来源几乎不变 |
| tags | 24h | 标签几乎不变 |
| bookmarks | 5min | 用户可能随时操作 |
| reports | 5min | 每日定时更新 |
| stats | session | 页面关闭即失效 |

### P-1.4 CDN / 静态资源缓存

**现状**：每次构建后用户可能需要硬刷新才能看到新页面

**方案**：
- Vite 已支持 hash 文件名（如 `index-B8KcXxrC.js`）
- Nginx 配置 `immutable` 缓存头：

```nginx
location /assets/ {
    expires 1y;
    add_header Cache-Control "public, immutable";
}
```

## 四、体验完善（建议优先级：中低）

### E-3.1 Toast 通知系统

**现状**：收藏/取消收藏操作无任何视觉反馈

**方案**：

```jsx
// components/Toast.jsx
export function useToast() {
  const [toasts, setToasts] = useState([]);

  const toast = useCallback((message, type = 'info', duration = 2000) => {
    const id = Date.now();
    setToasts(prev => [...prev, { id, message, type }]);
    setTimeout(() => setToasts(prev => prev.filter(t => t.id !== id)), duration);
  }, []);

  return { toasts, toast };
}
```

| 操作 | Toast 文案 | 类型 |
|------|-----------|------|
| 添加收藏 | 已收藏 ✓ | success |
| 取消收藏 | 已取消收藏 | info |
| AI 回复出错 | AI 回复失败，请重试 | error |
| 登录成功 | 登录成功 ✓ | success |
| 操作失败 | 操作失败，请重试 | error |

### E-3.2 空状态设计

**现状**：收藏/历史/搜索结果/筛选无结果时，显示空白或简单的"暂无内容"

**方案**：

| 页面 | 空状态文案 | 行动按钮 |
|------|-----------|---------|
| 收藏 | 还没有收藏过文章 | 去看看今日日报 → |
| 浏览历史 | 还没有浏览记录 | 去看看今日日报 → |
| 搜索结果 | 没有找到相关文章 | 换个关键词试试 |
| 筛选无结果 | 没有匹配的文章 | 清除筛选条件 |
| 归档日历（无日报月） | 该月没有日报 | 返回最新日报 → |

### E-3.3 Error Boundary

**现状**：某组件崩溃 → 整个页面白屏

**方案**：

```jsx
// components/ErrorBoundary.jsx
class ErrorBoundary extends React.Component {
  state = { hasError: false, error: null };
  static getDerivedStateFromError(error) { return { hasError: true, error }; }

  render() {
    if (this.state.hasError) {
      return (
        <div className="p-8 text-center">
          <p style={{ color: 'var(--color-high)', fontSize: '14px' }}>该区域加载失败</p>
          <button onClick={() => this.setState({ hasError: false })}>重试</button>
        </div>
      );
    }
    return this.props.children;
  }
}
```

包裹范围：
- 文章阅读器（独立 Error Boundary，不影响首页）
- AI 对话面板（聊天崩溃不影响文章阅读）
- 个人中心统计数据
- 搜索结果

# AI 批处理（核心 Token 消耗）

## 位置

`processor/ai_processor.py`

## 处理内容

每次 AI 调用同时完成四项任务：

```
一次 DeepSeek 调用
    ├── ① 中文摘要生成   (~200 tokens 输出)
    ├── ② 标签提取       (~50 tokens 输出)
    ├── ③ 重要性评分      (高/中/低)
    └── ④ 文章分类        (技术突破/产品发布/学术论文...)
```

## 批处理策略

```python
def process_articles(self, articles: List[Article]) -> List[Article]:
    # 按 10 篇一批分批
    batches = [articles[i:i+10] for i in range(0, len(articles), 10)]
    
    for batch in batches:
        result = self._process_batch(batch)
        for article, ai_data in zip(batch, result):
            article.summary = ai_data["summary"]
            article.tags = ai_data["tags"]
            article.importance = ai_data["importance"]
    return articles

def _process_batch(self, batch):
    messages = [
        {"role": "system", "content": SYSTEM_PROMPT},
        {"role": "user", "content": self._build_prompt(batch)}
    ]
    response = client.chat.completions.create(
        model="deepseek-chat",
        messages=messages,
        max_tokens=2048,
        temperature=0.1,  # 低温度 → 输出格式稳定
    )
    return self._parse_response(response)
```

## Token 消耗计算

| 项目 | 每批 (10篇) | 每次运行 (13-16批) | 每日 (3次) |
|------|------------|-------------------|-----------|
| 系统提示词 | ~300 | - | - |
| 文章内容 | ~2,200 | - | - |
| 输出 | ~2,000 | - | - |
| **合计** | **~4,500** | **~60,000-72,000** | **~216,000** |

## 为什么不一篇一篇调？

一次调用处理 10 篇 vs 10 次调用各处理 1 篇：

```
批处理：  1 次调用 × 4,500 tokens = 4,500 tokens
单篇：   10 次调用 × 1,200 tokens = 12,000 tokens
节省：                             63% 
```

## 面试话术

> "AI 处理层采用批处理策略，10 篇文章拼到一次 API 调用里，单篇成本约 450 tokens。一次调用同时完成摘要、标签、评分、分类四项任务，相当于四次任务合为一次。temperature 设为 0.1 确保输出格式稳定。如果一篇一篇调，成本会上涨 63%。"

## 关键经验

- **输出格式必须严格**：用 Pydantic 或 JSON mode 解析，否则 AI 格式漂移会整批报错
- **文章内容需要截断**：`raw_content[:800]` 限制单篇输入长度
- **失败重试**：单批解析失败时自动重试 1 次

# 自动化采集管道

## 架构全景

```
run.py (入口)
    │
    ├── 1. 加载配置 → config/sources.yaml
    │
    ├── 2. 采集 → 每个源独立 fetch()
    │       ├── arXiv        → HTTP API (XML)
    │       ├── HuggingFace  → HTTP API (JSON)
    │       └── 量子位/36氪等 → feedparser + 全文抓取兜底
    │
    ├── 3. 去重 → URL精确 → 标题相似度 → AI辅助
    │
    ├── 4. AI 处理 → 10篇/批 → 摘要+标签+重要性+分类
    │
    ├── 5. 生成日报 → 排序 + AI概览
    │
    └── 6. 写入 Supabase
```

## 设计模式：策略模式

所有采集器继承 `BaseCollector`，实现同一个 `collect()` 接口：

```python
class BaseCollector(ABC):
    @abstractmethod
    def collect(self) -> List[Article]:
        pass

# 三个子类各自实现
class RSSCollector(BaseCollector): ...
class ArxivCollector(BaseCollector): ...
class HFCollector(BaseCollector): ...
```

统一输出 `Article` 数据结构：

```python
@dataclass
class Article:
    title: str
    url: str
    source_name: str
    raw_content: str          # 正文（RSS 全文 or arXiv 摘要）
    published_at: datetime
    summary: Optional[str]    # AI 处理后填充
    tags: List[str]           # AI 处理后填充
    importance: Optional[str] # AI 处理后填充
```

## RSS 采集 + 全文抓取（核心）

```python
class RSSCollector(BaseCollector):
    MIN_CONTENT_LENGTH = 100

    def collect(self) -> List[Article]:
        feed = feedparser.parse(rss_url)
        for entry in feed.entries:
            content = entry.get("summary", "")
            # 摘要太短 → 自动抓原文
            if len(content) < self.MIN_CONTENT_LENGTH:
                full = self._fetch_full_content(entry.link)
                content = full or content
            articles.append(Article(...))
        return articles
```

## arXiv API 采集

```python
class ArxivCollector(BaseCollector):
    BASE_URL = "http://export.arxiv.org/api/query"

    def collect(self) -> List[Article]:
        # 限流：arXiv 要求 3 秒 1 次
        self._rate_limit()

        query = f"search_query=cat:cs.AI&sortBy=submittedDate&max_results=50"
        xml_data = urllib.request.urlopen(f"{self.BASE_URL}?{query}").read()

        root = ET.fromstring(xml_data)
        for entry in root:
            title = entry.find('title').text
            summary = entry.find('summary').text  # arXiv 直接带摘要
```

## 三层去重管道

```
185 篇 → URL精确去重 → 156 篇
         → 标题相似度去重 (阈值85%) → 156 篇
         → AI辅助去重 (70%-85%)    → 不变
```

标题相似度计算：

```python
from difflib import SequenceMatcher
similarity = SequenceMatcher(None, title_a, title_b).ratio()
# > 0.85 → 自动合并为同一事件
# 0.70-0.85 → 交给 AI 判断
```

## 面试话术

> "采集层采用策略模式，每个信息源实现独立的 Collector 类，统一输出 Article 对象，新增源只需要写一个类。单个源出错会优雅降级，不影响全局。RSS 摘要太短时自动抓取原文全文，确保 AI 处理时有足够信息量。配置驱动设计使得加新源只改 yaml 不动代码。"

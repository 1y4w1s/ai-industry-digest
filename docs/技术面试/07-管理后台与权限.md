# 管理后台与权限

## 入口

`/admin` → 需要 `role = 'admin'` 权限

## 权限模型

```sql
user_profiles 表
  id: UUID (PK, FK → auth.users)
  role: TEXT DEFAULT 'user'  -- 'user' | 'admin'
  created_at: TIMESTAMP
  updated_at: TIMESTAMP
```

## 管理员校验流程

```
访问 /admin
    │
    ▼
前端：检查 Supabase session
    ├── 无 → 跳转 /login
    └── 有 → 调用后端 API
            │
            ▼
后端：get_current_admin()
    ├── verify_token(token) → user_id
    ├── 查 user_profiles.role
    ├── role == 'admin' → 返回管理面板
    └── role != 'admin' → 403 拒绝
```

## 后端权限校验

```python
# 可复用的管理员依赖注入
def get_current_admin(user_id=Depends(get_current_user)) -> str:
    profile = db.get_or_create_profile(user_id)
    if profile.get("role") != "admin":
        raise HTTPException(status_code=403, detail="需要管理员权限")
    return user_id

# 使用时只需一行
@app.get("/api/admin/stats")
def admin_stats(admin_id: str = Depends(get_current_admin)):
    return get_system_stats()

@app.post("/api/admin/cache/clear")
def clear_cache(admin_id: str = Depends(get_current_admin)):
    cache_service.clear()
    return {"status": "ok"}
```

## 管理后台功能

| 功能 | 实现 | 对应 API |
|------|------|---------|
| 系统概览 | 统计 API | `GET /api/admin/stats` |
| 热门文章 | 阅读量排行 | `GET /api/admin/popular` |
| 来源分布 | 各来源数量 | `GET /api/admin/sources` |
| 缓存管理 | 统计/清空 | `GET+POST /api/admin/cache` |
| 用户管理 | 用户列表 | `GET /api/admin/users` |

## 前端路由保护

```jsx
function AdminRoute({ children }) {
    const { user, loading } = useAuth();

    if (loading) return <Loading />;
    if (!user) return <Navigate to="/login" />;

    // 检查是否为管理员
    const [profile, setProfile] = useState(null);
    useEffect(() => {
        getProfile(user.id).then(p => setProfile(p));
    }, [user]);

    if (profile?.role !== 'admin') return <Navigate to="/" />;

    return children;
}

// 使用
<Route path="/admin" element={
    <AdminRoute>
        <AdminDashboard />
    </AdminRoute>
} />
```

## 面试话术

> "管理后台的权限模型简洁有效——user_profiles 表加 role 字段，后端用 FastAPI 的 Depends 依赖注入做校验。`get_current_admin()` 可复用，任何管理员接口只需加一个参数声明：`admin_id: str = Depends(get_current_admin)`。前端用 AdminRoute 组件包裹，自动检测 role 并拦截非管理员。"

## 设置管理员

```sql
UPDATE user_profiles SET role = 'admin'
WHERE id = '用户-UUID';
```

## 二、Signal 项目 — 严格提问（重点区域）

### 2.1 基础拷问

**Q1: "这个项目你做了多久？是一个人做的吗？"**
> ⚠️ 简历写 2026.05-2026.06，约 1 个月

**面试官想听的：** 诚实回答是一个人。说清楚哪些部分是 AI 辅助的。  
**雷区：** 说"1 个月做完"听起来很厉害但也容易被质疑深度。

**Q2: "你说的成功率 >95%，具体怎么定义的？成功率是用什么除以什么？"**
> ⚠️ 简历原文：成功率 >95%

**如果回答不清楚：** 面试官会觉得你在编数据。  
**正确回答：** "每次采集流程会尝试采集 N 篇文章，成功入库的 / 总的采集数。失败原因包括：源站超时、RSS 解析失败、AI API 超时。我加了一个重试机制（3 次退避重试），95% 的数据来自连续 2 周的观察。"

**Q3: "日均采集 100+ 篇，7 个源，每个源也就十几篇。这数据是稳定值还是峰值？"**
> ⚠️ 简历原文：日均采集 100+ 篇 AI 资讯，覆盖 7 个高质量信息源

**正确回答：** "是日均值。arXiv 每次 50-80 篇，RSS 源每次 20-40 篇，HF Daily 每次 10-15 篇。实际采集成功率约 85-90%，经过 AI 去重后最终入库 40-60 篇。100+ 是采集量的原始值。"

**Q4: "你写了 36 个 API — 每一个都有测试吗？测试覆盖率多少？"**
> ⚠️ 简历原文：36 个 RESTful API

**你的答案：** 41 个测试覆盖主要业务逻辑，但不是每个 API 都有单独测试。没测的部分手动测过。

### 2.2 技术深度拷问

**Q5: "你说主动进行安全重构（JWT 验签、XSS 防御），那修复之前是什么状态？"**

**可能追问链：**
- "JWT 不验签的话，攻击者怎么利用？" → 构造任意 JWT，设 sub 为别人 UUID
- "你在前端还是后端做的 XSS 过滤？" → 前端用 DOMPurify
- "为什么不在后端也做一道？" → LLM 回复信任源是后端，但可能被 prompt injection
- "DOMPurify 的原理是什么？" → 解析为 DOM 树，删掉非白名单节点，序列化

**Q6: "你说'在页面有大量数据的情况下也能保障加载速度流畅'，大量是多少？怎么衡量的？"**

**你的答案：** 首页一次展示约 40-60 篇文章。用 React.lazy 代码分割 + 后端分页 (range offset)。页面加载时间 < 2s。"大量数据"这个表述不够精确，改了更好。

**Q7: "前端 9 个页面，你的路由设计是怎么做的？权限控制呢？"**

**正确回答：** 用 React Router v6 嵌套路由。私有页面用 `<PrivateRoute>` 包裹，检查 `isLoggedIn()`，未登录跳 `/login`。知识库权限在 API 层用 `_doc_access_filter` 做 OR 逻辑（用户自己的文档 OR 公开文档）。

**Q8: "AI 个性化推荐具体是怎么工作的？"**

**回答框架：**
1. 用户行为 → TagExtractor（本地字符串匹配，零 API 成本）
2. tag → 写入 user_tags 表（权重累加）
3. 推荐时 → 按 user_tags 权重排序文章 → 返回 Top N
4. DeepSeek API 只用于摘要+重要性评分，不用来做推荐

### 2.3 运维拷问

**Q9: "你说'已上线公网稳定运行'，跑了多久？出过什么问题？"**

**你的答案：** 上线约 3 周。出过的问题：JWT 跳过验签上线 2 天发现后整改；一度没连接池导致 502 修成了单例。没有严重的宕机。

**Q10: "服务器配置是什么？能扛多少并发？"**

**你的答案：** 轻量云服务器，单核 2GB。没有做压力测试。如果上线正式业务，需要加负载均衡和水平扩展。目前用户量小，够用。

### 2.4 项目边界拷问

**Q11: "你同时写 Vue（熟悉）和 React（基础），但 Signal 用了 React，为什么不用你更熟悉的 Vue？"**

**Q12: "项目做了 1 个月，你觉得如果重做一遍，哪些地方会做得不一样？"**

## 三、聊天室项目 — 严格提问

**Q13: "你说的'本地消息队列 + 断线重连补发机制'是怎么实现的？"**

**回答框架：**
1. 消息队列：前端用数组暂存未发送消息，网络恢复后逐条发送
2. 断线重连：WebSocket 的 onclose 触发 → 指数退避重试（1s, 2s, 4s, 8s...）
3. 补发：重连成功后检查本地队列，发送之前失败的消> 
**追问：** "如果用户关闭页面再打开，未发送的消息怎么处理？" → 用 localStorage 持久化待发送队列

**Q14: "10+ 前端页面，你一个人写 UI 写了多久？用了 UI 库吗？"**

**回答：** 自己写的 CSS，没有用 Element UI 之类的库。适配日间/夜间双主题用 CSS 变量。

**Q15: "WebSocket 和 HTTP 有什么区别？你项目中什么场景用 WebSocket，什么场景用 HTTP？"**

**回答：** 聊天消息 → WebSocket（实时双工）。登录注册、用户资料 → HTTP（请求-响应模式）。

## 四、基础技能拷问（必问）

### 4.1 数据库

**Q16: "MySQL 的索引是怎么工作的？B+Tree 和 B-Tree 的区别？"**

**追问链：**
- "什么场景索引会失效？"
- "联合索引 (a, b, c) 查询 where b=1 能走索引吗？"
- "你项目里有哪些查询没建索引？"

**Q17: "MySQL 和 PostgreSQL 有什么区别？你更喜欢哪个？"**

**回答：** PostgreSQL 支持数组类型（tags TEXT[]）、GIN 索引、全文搜索（tsvector）。MySQL 生态更成熟。信号项目用 Supabase = PostgreSQL，因为 tags 是数组类型用 GIN 索引。

### 4.2 后端

**Q18: "Spring Boot 的 IoC 是什么？你怎么用的？"**

**追问链：**
- "@Autowired 和 @Resource 的区别？"
- "Bean 的作用域有哪些？"
- "你的项目里事务怎么控制的？"

**Q19: "FastAPI 和 Spring Boot 的区别是什么？"**

**回答：** FastAPI 轻量、异步原生、Python 生态适合 AI 集成。Spring Boot 重型、Java 生态适合企业级。Signal 选 FastAPI 是因为要调 DeepSeek API（Python SDK 成熟）。

### 4.3 前端

**Q20: "Vue 的响应式原理是什么？"**

**追问链：**
- "Vue2 和 Vue3 有什么区别？"
- "你知道 Vue3 的 ref 和 reactive 区别吗？"
- "组件通信方式有哪些？"

**Q21: "React 的 useState 和 useRef 有什么区别？"**

**回答：** useState 变化触发重新渲染，useRef 变化不触发。阅读器里追踪滚动百分比用 useRef（不需要渲染），搜索条件用 useState（需要渲染新结果）。

### 4.4 部署与工具

**Q22: "GitHub Actions 的 workflow 文件结构是什么样的？"**

**回答：** `name: on: [push/schedule] → jobs: [runs-on + steps]`。有三个 workflow：daily.yml（定时 6/12/18 点采集）、deploy.yml（push 自动部署）、backup.yml（每周备份）。

**Q23: "Nginx 反向代理配置怎么写的？为什么 try_files 要写 $uri $uri/ /index.html 三个参数？"**

**回答：** SPA 单页应用。用户访问 `/knowledge`，Nginx 找 `knowledge` 文件→没有→找 `knowledge/` 目录→没有→返回 `index.html`，前端 React Router 解析路由。少了 `/index.html` 的话直接返回 404。

## 一、OpenSpec 系列（项目管理）

### 1️⃣ openspec-explore — 探索与讨论

```
  用在哪：        项目初期，想法还不成熟，需要讨论的时候
  怎么说：        "我想探讨一下..." / "帮我分析一下..."
  AI 会做什么：    陪你聊天、画图分析、挑战你的想法
  会不会写代码：   ❌ 不会，只聊天不写代码
```

### 2️⃣ openspec-propose — 生成项目方案

```
  用在哪：        想法成熟了，要生成正式的项目方案和任务清单
  怎么说：        "帮我做个项目方案..."
  AI 会做什么：    生成 proposal.md（提案）+ design.md（设计）+ tasks.md（任务）
  会不会写代码：   ❌ 不会，只生成文档
```

### 3️⃣ openspec-apply-change — 开始写代码

```
  用在哪：        方案定好了，要正式开始实现功能
  怎么说：        "开始实现吧" / "帮我把代码写了"
  AI 会做什么：    按 tasks.md 的任务清单，逐个写代码
  会不会写代码：   ✅ 会，这是写代码的模式
```

### 4️⃣ openspec-archive-change — 归档完成

```
  用在哪：        一个功能做完了，要归档保存
  怎么说：        "这个做完了，归档吧"
  AI 会做什么：    把项目文件归档到 archive 目录
  会不会写代码：   ❌ 不会
```

### 5️⃣ openspec-sync-specs — 同步规格文档

```
  用在哪：        开发过程中需求变了，要更新规格文档
  怎么说：        "把改动同步到规格文档"
  AI 会做什么：    对比新旧需求，更新 spec 文件
  会不会写代码：   ❌ 不会
```

## 二、质量保障系列

### 6️⃣ TRAE-code-review — 代码审查

```
  用在哪：        写完代码后检查质量
  怎么说：        "帮我 review 一下代码"
  AI 会做什么：    检查代码质量、找 Bug、提改进建议
```

### 7️⃣ TRAE-security-review — 安全审查

```
  用在哪：        检查代码有没有安全漏洞
  怎么说：        "帮我做安全审查"
  AI 会做什么：    找 SQL 注入、XSS、密钥泄露等安全问题
```

### 8️⃣ grill-me — 压力测试方案

```
  用在哪：        方案做完了，想看看有没有漏洞
  怎么说：        "来拷打一下我的方案"
  AI 会做什么：    疯狂提问，逼你思考方案的薄弱环节
  用户原话：       "我选择被拷打"
```

## 三、设计系列

### 9️⃣ frontend-design — 前端设计

```
  用在哪：        要做网页/组件的 UI 设计
  怎么说：        "帮我设计一个页面..."
  AI 会做什么：    生成漂亮的前端页面，拒绝 AI 模板感
```

### 🔟 ui-ux-pro-max-skill-main — 网页 UI 制作

```
  用在哪：        要做网页 UI
  怎么说：        "帮我做个网页 UI..."
  和 frontend-design 的区别：    这个是 UI 为主
```

## 四、工具系列

### 1️⃣1️⃣ skill-creator — 创建新技能

```
  用在哪：        你想让 AI 学会做一个新的技能
  怎么说：        "帮我创建一个新技能..."
  AI 会做什么：    生成技能所需的全部文件
```

## 五、快速对照表

```
  你想做什么                           → 用哪个
  ─────────────────────────────────────────────────────
  项目初期，想法不成熟，想聊聊           → explore
  想法成熟了，生成方案和任务             → propose
  开始写代码实现                        → apply-change
  做完了，归档                          → archive-change
  需求变了，更新文档                    → sync-specs
  检查代码质量                          → code-review
  检查安全漏洞                          → security-review
  方案怕有漏洞，求拷打                  → grill-me
  做前端页面/UI 设计                   → frontend-design
  做网页 UI                            → ui-ux-pro-max
  创建新技能                            → skill-creator
```

## 六、一句话速记

```
  先探索   → 再提案   → 再实现   → 最后归档
  explore   propose   apply     archive
   聊天     出文档     写代码     保存
```

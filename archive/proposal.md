## 一、项目背景与动机

### 1.1 痛点

- AI 领域资讯爆炸，每天数十篇论文、产品发布、融资消息散落在不同平台
- 从业者需要花大量时间浏览多个信息源才能掌握行业动态
- 现有日报产品要么收费，要么内容泛泛、缺乏深度分析
- 信息源之间的重复报道浪费阅读时间

### 1.2 目标

构建一个**全自动、零人工维护、成本趋近于零**的 AI 行业日报系统：

- 每日固定时间自动收集多个信息源的文章
- 通过 AI 生成摘要、标签分类、重要性排序
- 提供趋势分析和行业洞察
- 支持用户登录、收藏、浏览历史、PDF 导出等个人化功能
- 以简洁美观的网页形式呈现

## 二、核心理念

```
  信息收集         AI 处理         价值输出           用户价值
  ──────────────────────────────────────────────────────────────
  广度覆盖    →   智能筛选     →   可消费的日报    →   收藏/历史/PDF
  不遗漏      →   去重/摘要    →   有深度/有观点  →   个人知识库
  自动化      →   低成本      →   零人工维护     →   持续积累
```

## 四、产出物规格

### 4.1 日报结构

```
AI 行业日报 · YYYY-MM-DD

📊 今日概览
  - 收录 N 篇文章，覆盖 M 个信息源
  - 热点关键词：[tag1, tag2, tag3]
  - 重点事件简述

💡 分析师点评
  - 今天最值得关注的趋势/事件
  - 简短的理由说明

📰 文章列表（按重要性排序）
  🔴 高重要性 (N 篇)    ☆ 收藏按钮
    - 标题 | 摘要 | 标签 | 来源 | 原文链接 | 👍/👎（登录后可用）
  🟡 中重要性 (N 篇)
    ...
  ⚪ 低重要性 (N 篇)
    ...

📈 趋势信号（可选）
  - 近几天某方向论文/新闻激增
```

### 4.2 用户功能清单

| 功能 | 说明 | 是否需要登录 |
|------|------|------------|
| 日报浏览 | 查看每日聚合报告 | ❌ 不需要 |
| 搜索/过滤 | 按关键词、标签、日期检索 | ❌ 不需要 |
| 收藏文章 | 保存感兴趣的文章到个人收藏夹 | ✅ 需要 |
| 浏览历史 | 自动记录看过的文章 | ✅ 需要 |
| PDF 导出 | 单篇文章导出为 PDF | ❌ 不需要（纯前端实现）|
| 文章反馈 | 👍/👎 评价摘要质量 | ✅ 需要 |

### 4.3 AI 处理输出格式

```json
{
  "summary": "不超过 150 字的中文摘要，保留关键信息",
  "tags": ["融资", "产品发布"],
  "importance": "high",
  "reason": "头部公司重大产品发布，影响行业格局"
}
```

## 五、约束与前提

- 后端服务器配置极低，**不能本地跑大模型**
- 全部使用免费/低成本服务搭建
- 已有 DeepSeek-V4 API 额度（月费约 ¥2），作为 AI 处理主力
- 目标用户是**国内中文读者**，登录方案需在国内可无障碍使用
- 正式前端最后阶段开发，前期用简陋页面验证后端

## 六、风险与缓解措施

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|

| 免费数据库超限 | 低 | 中 | 预估年使用约 180MB，免费层 500MB 足够 |
| Supabase Auth 邮件进垃圾箱 | 中 | 中 | 采用"邮箱+密码"主力登录，GitHub OAuth 辅助，找回密码邮件用备用文案 |

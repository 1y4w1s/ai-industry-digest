# Signal — AI 行业日报聚合平台 · 简历项目总结

> **项目地址**: http://43.139.133.245:8080  
> **技术栈**: React + FastAPI + Supabase + GitHub Actions  
> **一句话**: 全栈自动化 AI 资讯平台，采集→AI 处理→日报生成→前端展示全链路自动化

---

## 一、技术栈

### 前端

| 技术 | 用途 | 熟练度 |
|------|------|--------|
| **React 18** | 组件化 UI 构建 | Hooks + Context + 函数式组件 |
| **Vite** | 前端构建工具（秒级热更新） | 开发/生产构建 |
| **Tailwind CSS v4** | 原子化 CSS 框架 + CSS 变量主题系统 | 响应式布局 + 亮暗主题 |
| **React Router v6** | 客户端路由（嵌套路由 + 参数传参） | 10+ 路由页面 |
| **html2canvas + jsPDF** | 文章导出 PDF（自动分页） | 客户端渲染截图 |
| **SpeechSynthesis API** | 文章朗读 TTS | 中文语音分段朗读 |

### 后端

| 技术 | 用途 | 熟练度 |
|------|------|--------|
| **Python FastAPI** | RESTful API 框架 | 异步路由 + 依赖注入 + 认证中间件 |
| **Supabase (PostgreSQL)** | 数据库 + Auth 认证 | RLS 策略 + JWT + GitHub OAuth |
| **httpx** | 异步 HTTP 客户端 | 信息源采集 |
| **feedparser** | RSS 解析 | 多源标准化采集 |
| **BeautifulSoup** | HTML 内容清洗 | 提取正文 |
| **DeepSeek API** | AI 文本处理 | 摘要生成 + 标签提取 + 重要性评分 + 实体识别 + 关系抽取 |

### DevOps

| 技术 | 用途 | 熟练度 |
|------|------|--------|
| **GitHub Actions** | CI/CD | 自动部署 + 定时采集（3 次/天） |
| **Nginx** | 反向代理 + 静态文件服务 | 前后端统一端口代理 |
| **SCP** | 自动化代码同步 | GitHub → 服务器推送式部署 |
| **Linux (Ubuntu)** | 生产环境 | uvicorn 进程管理 + 后台持久运行 + cron 定时任务 |

## 二、核心功能清单

### 2.1 自动化内容流水线
- 每天自动采集 arXiv、RSS 等多源 AI 行业资讯（共 10+ 信息源）
- 基于 DeepSeek API 实现：**AI 摘要生成**、**标签分类**、**重要性评估**（高/中/低）
- 自动生成**每日日报**，按来源分组展示，AI 精读置顶
- 全流程通过 GitHub Actions 定时运行（**6:00 / 12:00 / 18:00**）

### 2.2 前端功能
- **日报首页**：日期导航 → 文章卡片列表（来源分组 + 重要性标记）
- **文章阅读器**：AI 精读摘要 + 原文展示 + 深入 AI 对话 + TTS 朗读 + PDF 导出
- **全文搜索**：关键词搜索 + 标题/摘要**高亮标记** + AI 推荐面板
- **收藏系统**：收藏/取消收藏 + 分页列表
- **浏览历史**：按日分组 + 计数 + **一键清除全部**
- **个人中心**：阅读统计 + 收藏数 + 连续天数 + 月度**热力图** + 阅读**趋势折线图**
- **日报归档日历**：按月浏览历史日报，点击直接跳转
- **设置**：亮/暗/跟随系统主题 + 小/中/大字体
- **用户认证**：邮箱注册/登录 + GitHub OAuth
- **移动端适配**：响应式布局（375px+） + 侧栏抽屉 + 横向滚动筛选

### 2.3 知识库管理
- **文档管理**：上传 TXT/MD/PDF/DOCX → 自动切片 → AI 实体识别 + 关系抽取 → 知识图谱
- **知识图谱**：Canvas 力导向图可视化，支持交互式悬浮查看实体详情
- **语义搜索**：支持按文件名实时搜索、按类型/状态/来源多条件筛选
- **批量操作**：批量删除/处理/下载文档
- **自动扩容**：每天 cron 定时从日报文章自动导入知识库，`scripts/import_to_kb.py` 支持全量/增量/日期范围导入
- **API**：13 个 RESTful 端点，覆盖文档 CRUD、预览、下载、标签编辑、批量操作

### 2.4 基础架构
- **Supabase 数据库**：RLS 行级安全策略保护用户数据
- **JWT 认证**：API 请求自动校验
- **缓存策略**：localStorage 前端缓存（sources / tags / bookmarks / history / stats），减少 API 请求
- **单元测试**：pytest 覆盖数据库层和 API 层（**20 项测试**）

## 三、架构亮点

### 3.1 全自动化部署
```
git push → GitHub Actions checkout → SCP 同步代码到服务器
  → pip install → npm run build → 重启 uvicorn 后端
```
- 无需手动 SSH 登录部署
- 部署时间约 **2 分钟**

### 3.2 数据流水线
```
信息源 (arXiv/RSS) → 采集器 → 去重 → AI 处理（摘要/标签/重要性）
  → 日报生成 → Supabase 数据库 → FastAPI → React 前端
```
- 新增信息源只需在 `config/sources.yaml` 添加配置
- AI 处理失败时有降级策略（不阻塞整条流水线）

### 3.3 前端架构
- **自定义 Hooks**：`useReport`（日报数据）、`useFilter`（文章筛选）、`useTTS`（语音朗读）
- **CSS 变量主题系统**：亮/暗色模式全局切换，无 UI 闪烁
- **无第三方 UI 库**：所有 UI 组件（Select、Pagination、FilterBar、DateNav）纯自建

## 优先级 P0 — 核心体验必做

### P0-1：搜索结果独立页面

**问题：** 当前搜索后结果内联在 Home.jsx 中，Home 同时承担日报+搜索两种职责，代码耦合。

**方案：**
- 创建 `pages/SearchPage.jsx`，路由 `/search?q=xxx`
- Header 搜索回车后跳转到 `/search?q=xxx`
- SearchPage：全屏搜索结果，无右侧面板，聚焦搜索本身
- Home.jsx 不再处理搜索模式（可移除 `searching`/`searchResults`/`doSearch` 等约 40 行代码）

**估时：** 1h

### P0-2：阅读模式 UI 对齐新设计

**问题：** ArticleReader 是 v3.0 早期写的，配色/字体/间距与当前首页不一致。

**方案：**
- 检查并统一所有色值（`#1A1C1E`/`#2C2E32`/`#686C72`/`#8C9096`）
- 统一字体（标题 Source Serif 4，正文 Inter）
- 统一间距（4px 圆角，16-20px 内边距）
- 操作栏统一到底部

**估时：** 0.5h

### P0-3：品牌名清理

**问题：** 项目名、HTML title、favicon 仍为 AI Industry Digest。

**方案：**
- `index.html` title → "Signal — AI Industry Digest"
- 代码注释、API 描述中的旧名称更新
- favicon 替换（纯色 S 字母或留空）

**估时：** 0.3h

## 优先级 P1 — 体验提升

### P1-1：GitHub 仓库公开

**当前状态：** 仓库名仍为 `ai-industry-digest`，README 未更新。

**方案：**
- 仓库重命名为 `signal` 或保留 `ai-industry-digest`
- README 更新：新品牌名 + 截图 + 功能列表 + 技术栈
- 添加 LICENSE
- Verce l/ 部署配置

**估时：** 0.5h

### P1-2：移动端适配

**问题：** 当前移动端体验未经过专门打磨。

**方案：**
- 首页文章列表在 <768px 时全宽、去掉左右 padding 过大的问题
- ArticleGroup 在移动端的边距
- DateNav 在小屏幕的溢出处理
- 侧边栏：汉堡菜单弹出动画优化

**估时：** 1h

### P1-3：键盘快捷键

**方案：**
- `Cmd+K` / `Ctrl+K`：聚焦搜索
- `Escape`：关闭搜索 / 弹窗
- `h`：回到首页
- `r`：刷新当前日报

**估时：** 0.5h

## 优先级 P2 — 功能扩展

### P2-1：收藏/历史页面

**问题：** API 已存在（`/api/auth/bookmarks`、`/api/auth/history`），前端无页面。

**方案：**
- 创建 `pages/Bookmarks.jsx` — 收藏列表页
- 创建 `pages/History.jsx` — 浏览历史列表页
- 侧边栏导航项链接到对应页面
- 阅读模式内增加收藏按钮（空心/实心切换）

**估时：** 1.5h

### P2-2：AI 周报生成

**方案：**
- 后端：新增每周自动聚合趋势 + 生成周报
- 前端：周报列表页 + 周报展示（与日报类似但时间跨度为一周）
- 对于面试展示很加分（展示自动化运营能力）

**估时：** 2h（后端 1h + 前端 1h）

### P2-3：文章分享卡片

**方案：**
- Canvas API 生成分享卡片
- 卡片包含：标题、来源、摘要、品牌 Logo
- 支持下载为图片

**估时：** 1h
